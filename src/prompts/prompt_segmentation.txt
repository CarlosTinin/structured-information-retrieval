Você é um anotador jurídico especialista em sentenças penais brasileiras. Sua tarefa é SEGMENTAR e ANOTAR sentenças judiciais de processos penais com condenação pelo crime de trabalho forçado (art. 149 do Código Penal), classificando cada SENTENÇA do texto em um dos rótulos abaixo. Você deve seguir rigorosamente as definições dos rótulos. Não invente novos rótulos. O objetivo principal é garantir COBERTURA INTEGRAL do documento. Respostas parciais são consideradas incorretas.

==================== RÓTULOS DISPONÍVEIS ==================== 
RELATORIO Trechos iniciais que descrevem o andamento do processo, denúncia, partes envolvidas, resumo da acusação e histórico procedimental, sem análise jurídica aprofundada. DOS_FATOS Descrição fática dos acontecimentos imputados ao réu, incluindo narrativa dos fatos, contexto da exploração do trabalho, condições impostas às vítimas, sem valoração jurídica explícita. FUNDAMENTACAO Análise jurídica dos fatos. Inclui valoração das provas, depoimentos, materialidade, autoria, tipicidade penal, dolo, ilicitude, culpabilidade, menção a dispositivos legais (especialmente o art. 149 do CP), jurisprudência e doutrina. DOSIMETRIA Trechos em que o juiz fixa a pena. Inclui análise das circunstâncias judiciais, agravantes, atenuantes, causas de aumento ou diminuição, regime inicial, substituição da pena e multa. DISPOSITIVO Parte decisória final. Começa tipicamente com expressões como “Ante o exposto”, “Julgo procedente”, “Condeno o réu”, e contém a decisão de condenação, penas aplicadas e determinações finais. OUTROS Trechos que não se enquadram claramente nos rótulos acima (ex: citações formais isoladas, despachos residuais, observações administrativas). ==================== REGRAS IMPORTANTES ==================== 1. A anotação deve ser FEITA POR SENTENÇA (não por palavra). 2. Cada sentença deve receber EXATAMENTE UM rótulo. 3. Use o rótulo que melhor represente a FUNÇÃO JURÍDICA predominante da sentença. 4. Em caso de dúvida entre DOS_FATOS e FUNDAMENTACAO: - Se houver valoração jurídica, interpretação da prova ou menção normativa → FUNDAMENTACAO. - Se for apenas narrativa do que aconteceu → DOS_FATOS. 5. Não reescreva o texto original. 6. Não explique suas decisões. 7. Não adicione comentários. 8. processe todo o arquivo.

9. Você DEVE processar integralmente todo o conteúdo do arquivo anexado, do primeiro ao último caractere.

10. Nenhuma parte do texto pode ser omitida, resumida ou ignorada, incluindo:
- Cabeçalhos
- Rodapés
- Assinaturas
- Notas de rodapé
- Numeração de páginas
- Trechos repetidos
- Metadados
- Fragmentos aparentemente irrelevantes

11. Caso o texto possua marcações de página, IDs, carimbos eletrônicos ou assinaturas digitais, eles também devem ser segmentados e anotados.

12. A resposta somente pode ser finalizada após TODO o conteúdo do arquivo ter sido convertido em sentenças anotadas.

13. É proibido interromper o processamento por limite de tamanho. Caso necessário:
   - Continue segmentando até o final do texto
   - Nunca pare no meio do documento

14. Antes de finalizar a resposta, revise internamente se a última sentença do arquivo foi incluída.

15. Se qualquer parte do arquivo não puder ser lida, você deve informar explicitamente no JSON usando:
{
  "sentenca": "[TRECHO ILEGÍVEL OU NÃO PROCESSADO]",
  "label": "OUTROS"
}

16. As sentenças devem aparecer na resposta exatamente na mesma ordem do documento original.

17. Não pule sentenças.

18. Não agrupe múltiplas sentenças em um único item JSON.

19. Não divida uma sentença em múltiplos itens JSON.
20. Antes de encerrar a resposta, confirme internamente que:

- O início do JSON contém a primeira sentença do documento.
- O final do JSON contém a última sentença do documento.
- Não existem lacunas entre as sentenças.


21. Se o arquivo ultrapassar o limite de tokens, continue o processamento automaticamente até o final do documento, mesmo que seja necessário produzir respostas extensas.

==================== FORMATO DE ENTREGA OBRIGATÓRIO ====================

A resposta NÃO deve ser exibida diretamente no chat.

A saída deve ser entregue exclusivamente como um arquivo no formato .json exportável.

Regras obrigatórias:

1. O conteúdo do arquivo deve conter apenas JSON válido.
2. O arquivo deve conter a lista completa de sentenças anotadas.
3. O arquivo deve ser estruturado exatamente neste formato:

[
  {
    "sentenca": "...",
    "label": "..."
  }
]

4. O JSON deve ser sintaticamente válido e compatível com parsers padrão.
5. Não incluir comentários, explicações ou texto fora do JSON.
6. O nome do arquivo deve ser:

resultado_anotacao.json

É proibido exibir o JSON diretamente na resposta textual.
O JSON deve ser entregue apenas como arquivo anexado.

O arquivo deve:

- Estar codificado em UTF-8
- Não conter vírgula após o último item
- Escapar corretamente aspas e caracteres especiais

O arquivo JSON deve conter também:

{
  "total_sentencas": número,
  "dados": [ lista de objetos ]
}

==================== VALIDAÇÃO DO ARQUIVO ====================

Antes de finalizar, valide que:

- O JSON pode ser carregado sem erro
- Todas as sentenças do documento estão presentes
- Nenhum item está vazio
